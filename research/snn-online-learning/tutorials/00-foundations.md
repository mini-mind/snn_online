# 前置概念：从深度学习视角理解 SNN 在线学习

这份讲义面向“懂一点机器学习/工程，但不是神经动力学从业者”的读者。目标是让后续每篇论文教程都能读懂，不需要先系统学习计算神经科学。

## 1. 这条路线到底在反对什么

不是反对深度学习本身，而是反对把“反向传播 + 大批量离线训练”当作唯一学习范式。

普通深度学习训练大致是：

```text
收集数据
前向计算
计算 loss
反向传播梯度
优化器更新权重
重复很多 epoch
部署时冻结或很少更新
```

这套范式强大，但和你想要的系统有几处冲突：

- 在线性弱：部署后通常不持续改本体权重。
- 局部性弱：一个权重的更新依赖全网络反传来的梯度。
- 时序成本高：RNN/Transformer 处理长时序时，训练常需要保存大量历史状态。
- 持续学习困难：新数据更新容易破坏旧能力。
- 生物合理性弱：大脑不太可能保存完整计算图并反向传播精确梯度。

SNN 在线学习路线要问的是：

```text
能不能让每个突触只用局部活动 + 少量调制信号，就在交互过程中更新自己？
```

## 2. SNN 和普通神经网络有什么不同

普通 ANN 神经元输出连续数值：

```text
y = activation(Wx + b)
```

SNN 神经元更像一个带内部状态的动态系统：

```text
输入 spike 到来
膜电位上升
膜电位随时间泄漏
超过阈值时发出 spike
发放后重置或进入不应期
```

可以把 LIF 神经元粗略想成：

```text
v_t = leak * v_(t-1) + weighted_input_t
if v_t > threshold:
    spike_t = 1
    v_t = reset
else:
    spike_t = 0
```

这里的关键变化是：

- 神经元有状态，不只是静态函数。
- 信息通过离散 spike 事件传递。
- 时间本身是计算的一部分。
- 稀疏事件可以带来能耗优势。

## 3. 为什么 SNN 难训练

最大麻烦是 spike 不可导。

普通反向传播依赖“输出对输入的导数”。但 spike 是阈值事件：

```text
膜电位低于阈值 -> 0
膜电位超过阈值 -> 1
```

这个阶跃函数几乎处处导数为 0，在阈值处不可导。于是直接反传会失效。

常见解决路线有三类：

- surrogate gradient：前向仍用 spike，反向时用一个平滑近似导数。
- ANN-to-SNN：先训练普通 ANN，再转换成 SNN。
- local plasticity：不用全局反传，改用局部可塑性规则。

你的项目关注第三类：局部在线学习。

## 4. 什么是 eligibility trace

eligibility trace 可以翻译为“资格迹”或“可塑性痕迹”。它回答一个问题：

```text
某个突触最近是否参与了可能需要被奖励或惩罚的活动？
```

假设一个动作在 3 秒后才得到奖励。奖励到来时，系统需要知道哪些突触在 3 秒前参与了决策。eligibility trace 就是这个“临时标记”。

直觉上：

```text
pre neuron 发过 spike
post neuron 当时也接近发放或已经发放
这个突触对后续活动有影响
=> eligibility trace 增加

时间流逝
=> trace 衰减
```

所以它是一种短期记忆：

```text
e_ij(t) = decay * e_ij(t-1) + local_activity_tag
```

真正的权重更新通常等到调制信号出现：

```text
delta_w_ij = learning_rate * e_ij(t) * modulation(t)
```

## 5. 什么是三因子学习规则

两因子规则：

```text
delta_w = f(pre_activity, post_activity)
```

典型例子是 Hebbian learning 和 STDP。它们只看突触两端。

三因子规则：

```text
delta_w = f(pre_activity, post_activity) * third_factor
```

第三因子可以是：

- 奖励预测误差。
- 监督标签信号。
- 预测误差。
- 新奇性。
- 注意力。
- 内稳态压力。
- 安全惩罚。

这很重要，因为“参与了活动”不等于“应该增强”。第三因子给出方向和意义。

## 6. BPTT、e-prop、ETLP 的区别

BPTT：

```text
保存每个时间步的状态
把循环网络按时间展开
从未来误差往过去反传
得到较精确梯度
```

优点是效果强；缺点是不在线、不局部、存储重。

e-prop：

```text
每个突触在线维护 eligibility trace
网络接收学习信号
用 trace * learning_signal 近似梯度
```

它试图保留梯度下降的数学目标，同时避免完整 BPTT。

ETLP：

```text
用更简单的事件驱动三因子规则
pre spike trace + post membrane voltage + projected label
重点是硬件友好和在线实现
```

它更像“把局部规则做成能跑在 neuromorphic hardware 上的工程版本”。

## 7. 为什么只靠奖励不够

稀疏奖励的问题是：反馈太晚、太少。

例如 agent 走迷宫，只有到终点才奖励。如果它走了 500 步才成功，奖励到来时很难知道哪一步关键。

所以需要其他学习信号：

```text
预测误差：我以为下一状态是 A，结果是 B
新奇性：这个状态没见过
不确定性：模型不确定自己预测对不对
内稳态：活动太强/太弱，需要调节
安全惩罚：某些状态或动作必须抑制
```

这也是为什么阶段 2 的“预测学习 / 世界模型”很关键。它让系统在没有外部奖励时也能学习。

## 8. 世界模型和 dreaming 是什么

world model 学的是：

```text
当前状态 + 动作 -> 下一状态
当前状态 + 动作 -> 可能奖励
```

如果模型学得足够好，agent 可以在内部模拟：

```text
如果我做动作 a，会到哪里？
如果下一步再做 b，会怎样？
```

dreaming / replay 指的是用内部模型生成模拟经验，再用这些经验训练策略。它的工程意义是提高样本效率，减少真实环境交互成本。

## 9. 本项目应如何评价一个算法

一个算法越符合下面条件，越接近你的目标：

- 推理/交互时能更新本体权重。
- 更新不需要完整反向传播。
- 每个突触只依赖局部状态和少量调制信号。
- 能处理延迟奖励。
- 能在新任务上学习，同时不马上忘旧任务。
- 能用于循环网络。
- 能从预测误差、奖励误差、内稳态等多源信号学习。
- 能接入动作选择和控制任务。

反过来，如果一篇论文只是：

```text
用反向传播训练 SNN
把 ANN 转成 SNN
用 spike 提高推理效率
用 LLM 做稀疏推理
```

它可能仍有参考价值，但不是核心答案。
